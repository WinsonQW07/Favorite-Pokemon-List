Noteworthy

  1. Utilized the Bulma framework to make the app mobile friendly. The framework was used to give color and spacing to many elements of each page. The select lists and buttons were customized with Bulma to give them more style.
  2. The JavaScript code is written in ES6 modules which imports and exports appriopriate data.
  3. The user's favorited Pokemon are saved to the local storage as an array. The Pokemon are displayed within the Favorites tab. They are shown in multiple sections with up to 10 Pokemon being shown at once. The Pokemon can be removed from the list either individually or all together. Each Pokemon is created using a web component. Along with the Pokemon, the header and footer were also created using components.
  4. The app displays states that lets the user know that something is currently happening. The states consist of text that lets the user know that they must do something first and a loading gif that indicates something is trying to be displayed.
  5. Fetch was used to get data from the Pokemon API. The data included each of the 18 Pokemon types along with the data of each Pokemon.
  6. Firebase was utilized to save data from all users. Every Pokemon that is favorited by a user will be saved into the database. The most favorited Pokemon will be displayed in the "Community" tab.
  7. The "Community" tab has a button that plays music and shows a confetti animation. The button can be pressed to stop both of these features.
  8. Various buttons will change to show a loading animation to represent data being loaded in.
  9. The "Favorites" tab better shows the number of favorite Pokemon by showing the number of pages and what page the user is currently on.
  10. The app was modified so you no longer need to push a button to create a list of Pokemon after selecting a type. The button was modified to only change the list if there are any filters applied.
  11. User data within the app is saved to the local storage. Whenever the user comes back to the page, their previous filters, types, and Pokemon will be displayed.

Grading

  1. Fetch was used to return data from the API.
  2. The home page tells the user what the app does and what they can expect from the tabs. It also includes a button that randomly changes the image into a different type.
  3. The App has radio buttons and select lists to filter and display the Pokemon that the user wants. The favorite button allows the user to save the current Pokemon into the local storage.
  4. The Favorites page has ways for the user to delete every Pokemon or just one at a time. The user is able to press the next and previous buttons to view another group of Pokemon they have saved.
  5. The navbar will highlight and bold the current page that the user is on.
  6. Indicators have been created to let the user know that they may need to do something first or that something is loading.
  7. User data is saved to the local storage and returns it back to the user.
  8. Firebase was used for this project and saves the Pokemon's name and number of times it was favorited.

To Do

  1. The random button and images in the Home tab could be changed to a carousel. It would make it easier if a person wanted to see all of the types rather than trying to do it randomly or going to the App tab.
  2. In the App, it might help if there were more filter options. Alternatively, maybe add a function that lets the user view multiple Pokemon at once rathern than just one.
  3. In the Favorites tab, it might look nicer if each Pokemon had more data about themselves. The buttons could also be changed or arranged to show that these buttons do something while the other one is probably not one you want to carelessly press.
  4. More media queries should be utilized to make the app look better on smaller devicecs. The most notable change would probably be the font sizes of certain elements.
  5. The Favorites tab does not have too many indicators since the images load in by the time the user gets to the page. It might be worthwhile to add more indicators in case there are slow devices that may not immediately load in the images.
